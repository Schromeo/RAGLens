# RAGLens v0.4.0 - Local Release / Install & First-Run Experience

## Summary

RAGLens v0.4.0 focuses on local release readiness and first-run developer experience.

The goal is simple: a fresh external developer can clone the repo, start collector + dashboard with one clear path, generate reference traces, and inspect diagnostics quickly.

This release does not change warning engine behavior, trace contracts, storage schema, or dashboard data contracts.

## Highlights

- Docker Compose local stack for collector + dashboard
- Collector and dashboard Dockerfiles for repeatable local startup
- Persistent Docker volume for collector SQLite data
- Root `.env.example` for common local values
- Updated smoke-test and reset guidance for Docker and non-Docker paths
- README quickstart restructuring with a Docker-first path
- New reference app runbook for deterministic validation cases

## Docker Quickstart

From repository root:

```bash
docker compose up --build
```

In another terminal:

```bash
curl http://localhost:4319/health
```

Generate reference traces:

```bash
cd sdk/python
pip install -e .
python -m examples.reference_rag_app.run all
```

Open dashboard:

```text
http://localhost:5173
```

## Non-Docker Fallback Path

From repository root:

```bash
python scripts/start-raglens.py
```

Then generate reference traces:

```bash
cd sdk/python
python -m examples.reference_rag_app.run all
```

## Validation Commands

## Current Implemented Span Types

- retrieval
- llm

## Current Warning Rules

- no_retrieved_chunks
- low_retrieval_score
- duplicate_chunks
- weak_query_chunk_overlap
- numeric_mismatch
- conflicting_chunks
- answer_not_grounded

## Current Limitations

- Only retrieval and llm span types are implemented
- Python SDK distribution remains local editable install for now
- No LangChain/LlamaIndex adapters in this release
- No hosted/cloud/auth features
- No agent/tool/memory/retry spans
- No LLM-as-judge evaluator

## Not Included / Out of Scope

- New diagnostic rules
- Warning engine redesign
- SDK trace API changes
- Collector API contract changes
- Storage schema changes
- Dashboard data contract changes
- PyPI publishing
- LangChain or LlamaIndex adapters
- Hosted/cloud/auth product features

## Stability Notes

- v0.4.0 does not add new diagnostic rules.
- v0.4.0 preserves deterministic-first warning generation.
- v0.4.0 preserves the existing Python SDK trace contract.
- v0.4.0 is focused on first-run local developer experience.
